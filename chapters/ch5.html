QCA Extensions: mvQCA & tQCA

本章介紹 QCA 的兩個重要延伸：多值 QCA（mvQCA）允許條件取三個以上的離散值，突破清晰集的二元限制；時序 QCA（tQCA）則在布林最小化中加入條件出現的先後順序，使因果路徑更貼近「歷史過程」的現實。

5.1 多值 QCA（mvQCA） · Multi-Value QCA

清晰集 csQCA 只允許條件取 0（缺席）或 1（在場）。但社會科學的許多概念天然地有三個以上的離散類別——例如選舉制度（多數制、混合制、比例代表制），硬要把它壓縮成二元往往會損失重要資訊，甚至引起二元化偏誤（dichotomization bias）。

多值 QCA（mvQCA）允許條件取整數值 0, 1, 2, …（或更多），從而完整表示多類別概念，而不失去 QCA 集合論的核心邏輯。

本節研究案例

Hartmann & Kemmerzell (2008) 分析 30 個撒哈拉以南非洲國家的一黨禁令（party bans）。結果變項：BAN（是否存在一黨禁令，1=有，0=無）。條件包含：
RPG（統治政黨型態，3 個值）、MIL（軍事干預，2 個值）、GEO（地理位置，3 個值）、EIN（獨立後政治菁英，2 個值）、COL（殖民宗主國，3 個值）。

多值符號表示法 · Multi-value notation

在 mvQCA 的布林表達式中，多值條件用大括號和值表示：

符號意義對應 csQCA
RPG{0}RPG = 0（第一類）~RPG（缺席）
RPG{1}RPG = 1（第二類）RPG（在場）
RPG{2}RPG = 2（第三類）無對應
RPG{0,2}RPG = 0 或 2（合併類）部分對應
為何是大括號 { }？

在 mvQCA 的輸出中，RPG{1} 代表「RPG 等於 1 這個值」，而非清晰集的「RPG 在場」。這個符號體系來自多值邏輯（multi-valued logic），是 QCA 軟體（如 Thiem & Duşa 的 QCA 套件）的標準輸出格式。

5.2 mvQCA 資料準備與真值表 · Data Preparation & Truth Table

mvQCA 的資料格式與 csQCA 相似——仍是整數值，但允許超過 0/1 的範圍。每個多值條件取幾個值，就有幾個「位元」參與布林最小化。一個條件取 k 個值會產生 k 個二元對應變數（dummy expansion），讓 QCA 套件在內部計算。

載入資料 · Loading the HarKem dataset

> library(QCA) > data(HarKem) # Hartmann & Kemmerzell (2008) 資料集 > head(HarKem) ## RPG MIL GEO EIN COL BAN ## AN 0 0 1 1 2 0 ## BF 1 1 2 0 2 1 ## BN 0 0 0 1 2 0 ## BW 0 0 1 0 0 0 ## CM 1 1 2 1 2 1 ## CF 1 0 2 1 2 0 > sapply(HarKem, function(x) sort(unique(x))) # 查看各變項取值 ## $RPG: 0 1 2 $MIL: 0 1 $GEO: 0 1 2 ## $EIN: 0 1 $COL: 0 1 2 $BAN: 0 1
HarKem 變項說明

RPG：統治政黨型態（0=軍政府/一黨, 1=半競爭, 2=競爭）
MIL：軍事干預（0=無, 1=有）
GEO：地緣位置（0=西非, 1=中非, 2=東/南非）
EIN：菁英出身（0=殖民舊菁英, 1=新菁英）
COL：殖民宗主國（0=英國, 1=其他, 2=法國）
BAN：一黨禁令（0=無, 1=有）

建構 mvQCA 真值表 · Building the mvQCA truth table

建構 mvQCA 真值表的函數仍是 truthTable()，但需加上 type = "mv"（多值）。真值表的行數 = 每個條件取值數的乘積：本例 3 × 2 × 3 × 2 × 3 = 108 列。

> HarKemTT <- truthTable(HarKem, + outcome = "BAN", + conditions = c("RPG", "MIL", "GEO", "EIN", "COL"), + complete = TRUE, + show.cases = TRUE) > HarKemTT
mvQCA 的 incl.cut 預設值

mvQCA 的一致性（包含度）計算邏輯與 csQCA 相同（二元比例），因為條件和結果均為整數集合。incl.cut = 1 意味著一致性必須達到 100%（與 csQCA 相同）。如果允許略低於完全一致，可設 incl.cut = 0.8 並搭配 n.cut = 2（至少 2 個案例）。

5.3 mvQCA 布林最小化 · mvQCA Boolean Minimization

eqmcc() 搭配多值條件

> # 複雜解 > mvCS <- eqmcc(HarKemTT, details = TRUE, show.cases = TRUE) > mvCS ## M1: RPG{1}*MIL{1} + RPG{2}*GEO{2}*COL{2} + ... => BAN ## ## incl PRI cov.r cov.u ## M1 1.000 1.000 0.929 ... > > # 簡約解（加入邏輯餘項化簡） > mvPS <- eqmcc(HarKemTT, include = "?", + details = TRUE, show.cases = TRUE) > mvPS

解讀 mvQCA 輸出 · Interpreting mvQCA output

mvQCA 輸出的因果路徑使用大括號符號。以下是一個假設的解示例：

RPG{1}*MIL{1} + RPG{2}*GEO{2}*COL{2} ⇒ BAN
路徑 1：RPG{1}*MIL{1}
  → 統治政黨型態 = 半競爭（RPG=1）AND 軍事干預 = 有（MIL=1）
  → 這兩個條件共同出現 → 導致一黨禁令（BAN=1）

路徑 2：RPG{2}*GEO{2}*COL{2}
  → RPG = 競爭 AND 地處東/南非 AND 法國殖民地
  → 三條件聯集 → 導致一黨禁令
mvQCA 的 AND 與 OR 運算

與 csQCA 相同：* 代表 AND（集合交集），+ 代表 OR（集合聯集），波浪號 ~ 代表 NOT（補集）。但在多值條件下，取補集較複雜——~RPG{1} 代表「RPG 不等於 1」，等同於 RPG{0,2}（0 或 2）。

互動 mvQCA 符號解碼器

選擇條件取值，即時看對應的 mvQCA 大括號符號表示法，以及其在 csQCA 中的對應意義（若適用）。

5.4 時序 QCA（tQCA） · Temporal QCA

標準 csQCA / fsQCA 把所有條件視為無時序的同等成員——不管條件 A 在條件 B 之前或之後發生，它們都被等同對待。時序 QCA（tQCA）在布林最小化中加入條件的「先後順序」資訊，讓因果路徑反映歷史過程的時間維度。

本節研究案例

Ragin & Strand (2008) 以 19 世紀末至 20 世紀初，10 個國家建立農業結盟（「綠色聯合」）為例，研究三個條件的時序效果：
EBA（精英主導農業利益）、EBM（精英主導城市利益）、GRI（農民自主動員）
資料集：data(RagStr)（QCA 套件內建）。

tQCA 的核心概念：時序與組態 · Temporal sequences in QCA

tQCA 的關鍵貢獻是區分同樣的條件組合，但發生順序不同的情況。例如：

組態相同、順序不同 → 不同結果

A*B（A 先 B 後）≠ B*A（B 先 A 後）。傳統 QCA 把這兩種情況視為同一個組態「A 且 B」，tQCA 則將它們區分為不同的時序路徑。

時序組態的建立

每個案例除了記錄哪些條件「在場」，還記錄條件出現的先後時序（或同時出現）。這些資訊被編碼進時序真值表中。

時序路徑的解讀

最小化後的解以「>」表示時序先後：EBA > GRI 意味著「精英主導農業利益先出現，農民動員後出現」的時序路徑。

5.5 輔助條件與「無關」值 · Auxiliary Conditions & Don't-Care Values

tQCA 引入了兩個 csQCA 沒有的概念：輔助條件（auxiliary conditions）和「無關」值（don't-care / dc values）。

輔助條件 · Auxiliary conditions

在時序路徑中，某些條件的「存在或缺席」本身已由時序邏輯決定，不需要額外在場。這些條件被稱為輔助條件（auxiliary conditions, AC），用來確定時序中的「空白位置」。

「無關」值（dc 值） · Don't-care values

某個位置的條件值「對分析結果無影響」時，可編碼為「無關」（don't care）。在 R 的 QCA 套件中，「無關」值可以用以下方式表示：

寫法意義使用情境
-（破折號）此案例的此條件不適用資料矩陣中
dc（字串）don't care，與 - 同義部分函數輸入
負整數（如 -9）缺失或無關，由研究者設定數值型資料
NAR 語言的缺失值一般資料清理後
> data(RagStr) # Ragin & Strand 資料集 > RagStr ## EBA EBM GRI OUT ## AUS 1 1 0 1 ## BEL 1 1 0 0 ## CAN 0 1 1 -1 <-- 負整數 -1 表示 don't care (OUT) ## DEN 1 1 1 1 ## FIN 1 -1 1 1 <-- EBM = -1（don't care） ## FRA 1 1 0 0 ## GER 1 1 1 1 ## IRE 0 1 0 0 ## SWE 1 -1 1 1 <-- EBM = -1（don't care） ## USA 0 1 1 0 > # 確認「無關」值的設定 > RagStr[RagStr == -1] # 查看所有 dc 值的位置
RagStr 的時序意義

EBM = -1（FIN 與 SWE）的解讀：這兩個國家中，城市精英利益的存在/缺席對於農業聯盟的形成「無關緊要」——關鍵是農民動員（GRI=1）已先發生，城市精英的立場不再決定最終結果。這正是 tQCA 分析時序邏輯的精髓。

在 truthTable() 中處理 dc 值

> # tQCA 用 truthTable()，將 dc 值以 -1（或其他負數）標記 > RagStrTT <- truthTable(RagStr, + outcome = "OUT", + conditions = c("EBA", "EBM", "GRI"), + complete = TRUE, + show.cases = TRUE, + dcc = -1) # 指定 don't-care 代碼為 -1 > RagStrTT

5.6 tQCA 最小化 · tQCA Minimization

tQCA 的布林最小化邏輯與 csQCA 相同，但化簡時還需考慮「時序覆蓋 (temporal coverage)」：一個時序路徑只能覆蓋在正確時序下發生的案例。

> # 複雜解 > tCS <- eqmcc(RagStrTT, + details = TRUE, + show.cases = TRUE) > tCS ## M1: EBA*GRI + EBA*~EBM => OUT ## ## incl PRI cov.r cov.u cases ## EBA*GRI 1.000 1.000 0.600 0.400 DEN, FIN, GER, SWE ## EBA*~EBM 1.000 1.000 0.400 0.200 FIN, SWE > > # 簡約解（含邏輯餘項） > tPS <- eqmcc(RagStrTT, include = "?", + details = TRUE, + show.cases = TRUE) > tPS

解讀 tQCA 輸出 · Interpreting tQCA results

在 tQCA 中，解的表示與 csQCA 相同（AND = *、OR = +、NOT = ~）。時序關係體現在研究者對「哪個條件先出現」的理論說明中（通常在真值表建構時已編碼）。例如：

EBA*GRI ⇒ OUT（農業聯盟形成）
csQCA 讀法：精英主導農業（EBA）AND 農民動員（GRI）→ 聯盟形成

tQCA 讀法（搭配時序資料）：
  • FIN, SWE：GRI（農民動員）先於 EBA（精英動員）→ 聯盟形成
  • DEN, GER：EBA 與 GRI 同時或接近同時 → 聯盟形成

時序方向由原始資料的編碼決定，最小化的結果需搭配案例說明才有完整意義。
tQCA 的限制與爭議

tQCA 的時序資訊主要依賴研究者對案例的歷史判斷，而非自動從資料提取。若時序編碼不準確，分析結果可能有誤。另外，樣本數小時（tQCA 常見）邏輯餘項多，化簡效果有限。建議搭配深度個案研究（within-case analysis）驗證時序路徑的因果機制。

互動 tQCA 「無關」值模擬器

設定三個條件的值（0=缺席, 1=在場, dc=無關），觀察這個案例在真值表中的行為：是否能「覆蓋」多個真值表列。

5.7 三種 QCA 的比較 · Comparing csQCA, mvQCA, and tQCA

三種 QCA 變體各有其適用場景，選擇時應依據研究問題與資料特性：

面向 csQCA（清晰集） fsQCA（模糊集） mvQCA（多值） tQCA（時序）
條件型態 二元（0/1） 連續（0–1） 離散多值（0/1/2/…） 二元 + 時序
主要優點 概念簡潔、最小化穩健 捕捉程度差異、更貼近連續概念 避免二元化偏誤、保留多類別資訊 加入因果時序、反映歷史過程
主要限制 二元化損失資訊；容易過度化簡 閾值設定主觀性高；交叉點需理論支持 真值表列數急遽增加；邏輯餘項極多 時序資料難以系統化蒐集；小樣本問題
樣本數建議 10–50 個案例 10–150 個案例 通常需 >30 個案例 通常 <30 個案例（歷史比較）

本章回顧 · Chapter Summary

mvQCA 重點整理
  • 使用時機：條件天然有 3+ 個離散類別，且不宜二元化（避免資訊損失）。
  • 資料格式：整數值（0, 1, 2, …），與 csQCA 相同的 data.frame，但條件取多個值。
  • 真值表：truthTable() 自動偵測多值條件，行數 = 各條件取值數的乘積。
  • 輸出符號：大括號表示法 RPG{1}、COL{0,2}；AND/OR/NOT 符號與 csQCA 相同。
  • 主要挑戰：邏輯餘項急遽增多，限制化簡能力；建議搭配充分的案例數量。
tQCA 重點整理
  • 使用時機：因果過程有明確的時序邏輯，且研究者有可靠的歷史時序資料。
  • 「無關」值：以 -1（或其他負整數）在資料中標記 don't-care 值；用 dcc = -1 告訴 truthTable() 這些值的代碼。
  • 分析流程：與 csQCA 相同（truthTable() → eqmcc()），但需特別處理 dc 值。
  • 解讀：最小化結果本身不直接標示時序，需搭配原始時序資料和案例分析才能完整解釋因果路徑。
  • 限制：時序判斷高度依賴研究者的歷史知識；適合歷史比較研究，樣本數通常較小。
