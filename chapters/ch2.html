1. 安裝、套件與基本運算 · Install, packages, basics

R 可從官方網站 r-project.org 下載。它沒有像 SPSS 那樣的點選式介面——你用指令跟它溝通，指令打在主控台（console）裡，提示符號是 >。

「套件（package）」是擴充 R 功能的程式庫。QCA 套件就包含了做 QCA 的所有函式、資料集與說明檔。安裝與載入分兩步：

安裝（只需一次）／載入（每次開新 session）
> install.packages("QCA", dependencies = TRUE) > library("QCA")
為什麼要 dependencies = TRUE？

QCA 套件的部分功能依賴其他套件（例如 lpSolve）。設 dependencies = TRUE 會一併把這些相依套件裝好，省去後續報錯。

R 本質上是一台「可無限擴充的可程式化計算機」。最基本的工作流程是：寫一個指令 → 送給直譯器執行。結果前面的 [1] 是「索引」，代表這是輸出的第一個元素。

基本算術與邏輯
> 3 * 5 [1] 15 > sin(pi/2) [1] 1 > 6 == 7 # 是否相等 [1] FALSE > FALSE & TRUE # 邏輯 AND [1] FALSE > 2 == 2 | 2 == 3 # 邏輯 OR [1] TRUE
類別運算子 / 函式說明
算術+ - * / ^加、減、乘、除、次方
算術函式sum() prod() sqrt() abs() log() exp()總和、乘積、平方根、絕對值、對數、指數
取整round() floor() ceiling()四捨五入、向下、向上取整
比較== != > >= < <=等於、不等於、大小比較

3. 物件與賦值 · Objects & assignment

R 是物件導向語言：任何運算結果都能存成「物件」。最常用的賦值符號是「箭頭」<-（小於號加減號，中間不空格）。c() 函式把多個值串接成一個向量。

把兩個總和存進物件 x
> x <- c(sum(3, 5), sum(4, 9)) # 8 與 13 > x [1] 8 13
命名小撇步

因為 c 是內建函式，新物件最好別取名為小寫 c。命名可用駝峰式 dataAuthor 或點分隔 data.author。把整個運算式用括號包起來 (x <- ...)，可以「賦值的同時印出結果」。

4. 資料結構 · Data structures

社會科學最常用四種資料結構。它們的差別在於「維度」與「能不能混型別」：

  • 向量 Vector：單一列或欄，所有元素同型別。用 c()、rep()、seq() 或 1:5 產生。
  • 矩陣 Matrix：長方形排列，同型別。用 matrix()、cbind()、rbind() 建立，預設逐欄填入。
  • 資料框 Data frame：社會科學最常用，像矩陣但每一欄可不同型別。用 data.frame() 建立。
  • 清單 List：最有彈性，可把以上結構全裝進同一個物件。QCA 的 truthTable() 等複雜函式回傳的就是清單。
用三種變體的應用數做一個資料框
> QCAdat <- data.frame(variant = c("csQCA", "mvQCA", "fsQCA"), + number = c(170, 7, 62)) > QCAdat variant number 1 csQCA 170 2 mvQCA 7 3 fsQCA 62
向量循環（vector recycling）

當兩個長度不同的向量做運算時，R 會自動「循環」較短的向量來補齊（前提是長向量長度是短向量的整數倍）。這個特性在計算集合補集 1 - SET 時很方便——純量 1 不必重複寫成一整列。

5. 資料型別與階層 · Data types & hierarchy

最底層的物件分三種原子型別，依「優先序」由低到高為：logical（邏輯）→ numeric（數值）→ character（字元）。這個階層源自集合關係：邏輯值是實數的子集、實數又是字元的子集。

關鍵推論：在 R 裡 TRUE 等同數值 1、FALSE 等同 0。這正是稍後真值表能用數值運算來算「真假」的原因。可用 mode() 查型別、is.character() 檢查、as.integer() 強制轉換。

7. 集合運算 · Operations on sets

這節處理「集合的元素」運算（隸屬度假設為清晰）。四個核心函式：union()（聯集）、intersect()（交集）、setdiff()（差集）、setequal()（是否相等）。

注意「unique」

這些函式比較的是唯一元素。所以 A = c(1,2,3,4) 與 B = c(1,1,2,3,3,3,4) 用 setequal() 會回傳 TRUE（唯一元素相同）。若要檢查「完全相同」，要用 identical()。

8. 集合隸屬度運算 · Operations on set memberships

這是 QCA 計算的心臟。前一節算的是「集合裡有哪些元素」，這節算的是「每個案例對集合的隸屬度分數」。布林邏輯與隸屬度的對應規則只有三條：

布林運算意義隸屬度計算R 函式
AND（·）交集取最小值 minpmin()
OR（+）聯集取最大值 maxpmax()
NOT（小寫）補集1 - 隸屬度1 - SET
為什麼是 pmin / pmax 而不是 min / max？

pmin()／pmax() 取的是「平行最小／最大值」——它逐列（逐案例）比較，而不是把所有值丟在一起取單一最小值。每個案例各自算各自的，這才符合 QCA「每個案例有自己的隸屬度」的邏輯。

在資料框上計算複合運算式（取自原書示範）
> datFS$aBC <- pmin(1 - datFS$A, datFS$B, datFS$C) # a · B · C > datFS$"A+b+C" <- pmax(datFS$A, 1 - datFS$B, datFS$C) # A + b + C > datFS$"Ab+Bc" <- pmax( + pmin(datFS$A, 1 - datFS$B), + pmin(datFS$B, 1 - datFS$C)) # A · b + B · c
為什麼欄名要加引號？

像 "A+b+C" 這種含有布林「+」的欄名必須用雙引號包起來，否則 R 會把 + 當成算術加號，而不是欄位名稱的一部分。

10. 匯入匯出與求助 · I/O & getting help

  • 匯入：最簡單是把資料存成 tab 分隔的 .txt，再用 read.table("檔名.txt", header = TRUE, row.names = "State") 讀入。CSV 用 read.csv()；SPSS/Stata 檔可用 foreign 套件。
  • 注意路徑：Windows 路徑要寫成 "C:/folder" 或 "C:\\folder"，不能用單一反斜線。用 getwd() 看工作目錄、setwd() 切換。
  • 匯出：write.table(myfile, file = "myfile.txt", sep = "\t", quote = FALSE)。
  • 求助：在函式前加問號，例如 ?prod，會開啟說明文件（Description／Usage／Arguments／Details／Value／Examples）。
最常見的 NA 陷阱

很多函式（如 prod()、mean()）預設 na.rm = FALSE。只要資料裡有一個 NA，結果就會變成 NA。想忽略缺失值，要明確加上 na.rm = TRUE。準備資料時，所有缺失值都應先標成 NA、且欄名與數值都不要有空格。

本章重點回顧

① R 用指令溝通；install.packages("QCA") 再 library("QCA")。
② 四種資料結構：向量／矩陣／資料框／清單。
③ 型別階層 logical ⊂ numeric ⊂ character，所以 TRUE=1、FALSE=0。
④ QCA 計算三規則：AND→pmin、OR→pmax、NOT→1-x，逐案例計算。
⑤ 用這三規則就能手算真值表——正是 csQCA／fsQCA 的起點。
