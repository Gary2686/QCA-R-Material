3.1 校準清晰集 · Calibrating Crisp Sets

把「原始資料 (raw data)」轉換成「集合隸屬度分數」的過程稱為校準 (calibration)。清晰集的隸屬度只有 0（不屬於）或 1（屬於），校準就是在決定「切點」。

本章研究案例

Krook (2010) 分析 22 個西方民主國家的女性國會代表比例（依變項 WNP），條件為：選舉制度（ES）、婦女配額（QU）、福利國家類型（WS）、女性運動自主性（WM）、左翼政黨強度（LP）。本章複製她的 csQCA 分析。

類別變數的校準 · Categorical calibration

類別變數（如選舉制度）直接用 ifelse() 做條件判斷：

> Krook$ES <- ifelse(KrookRaw$es == "PR", 1, 0) # 比例代表制=1 > Krook$QU <- ifelse(KrookRaw$qu == "Yes", 1, 0) # 有配額=1 > Krook$WS <- ifelse(KrookRaw$ws == "SocDem", 1, 0)# 社民福利國家=1 > Krook$WM <- ifelse(KrookRaw$wm == "Autonom", 1, 0)# 自主型運動=1

連續變數的校準 · Continuous calibration

連續變數（如 lp 左翼政黨強度）需要先找「切點 (threshold)」，再用 calibrate() 轉換。找切點有三種方式：

視覺化分布

用 barplot() 或 density()（核密度圖）觀察資料分布，直觀判斷群集位置。

中位數 / 平均數

median(KrookRaw$lp) → 4.5；mean(KrookRaw$lp) → 4.55。作為備用參考。

階層聚類 findTh()

QCA 套件的 findTh() 以聚類分析自動建議切點。findTh(KrookRaw$lp) 建議 6；Krook 的理論判斷選 7。

> Krook$LP <- calibrate(KrookRaw$lp, thresholds = 7) # ≥7% = 1 > Krook$WNP <- calibrate(KrookRaw$wnp, thresholds = 30) # ≥30% = 1
calibrate() 的邏輯

對清晰集，calibrate(x, thresholds = t) 等同於 ifelse(x >= t, 1, 0)，但它統一了清晰集與模糊集的介面。預設 include = TRUE 代表「等於切點的值歸入集合（=1）」。

互動 清晰集校準模擬器

輸入一組數值與切點，即時看每個值被校準成 0 還是 1。

3.2 必要性檢定 · Testing for Necessity

必要性分析從觀察到結果出發，往回看條件是否在場。用兩個指標評估：

指標公式（清晰集）意義理想值
必要性 Inclusion
InclN		① ÷ (①+②)
= O=1 且 C=1 的案例 ÷ 所有 O=1 案例		「C 在場的頻率 vs 結果出現」；越高越支持 C 為必要≥ 0.9
必要性 Coverage
CovN		① ÷ (①+③)
= O=1 且 C=1 ÷ 所有 C=1 案例		必要條件是否「太過寬泛」（trivially necessary）；越高越好≥ 0.5–0.6

QCA 用 superSubset() 函式自動搜尋所有可能的條件（組合），找出符合 inclusion 與 coverage 門檻的必要條件集合：

> KrookNR <- superSubset(Krook, outcome = "WNP", + incl.cut = 0.9, cov.cut = 0.52) > KrookNR incl PRI cov.r --------------------------------- 1 wm+LP 0.909 0.909 0.667 2 WS+LP 0.909 0.909 0.909 ← coverage 最高 6 ES+LP 1.000 1.000 0.733 ← inclusion 最高 7 ES+WM 1.000 1.000 0.524 ...
superSubset() 的搜尋策略

它從最簡單的「單條件」開始，如果 inclusion 夠高，就向上組合；如果 inclusion 不夠，就嘗試聯集（OR）組合。完整算法確保不遺漏任何有潛力的結果。第 4 章 fsQCA 也是用同一函式。

互動 必要性 Inclusion & Coverage 計算器

輸入 2×2 格子的案例數，即時計算 InclN 與 CovN。

3.3 充分性檢定與真值表 · Sufficiency & Truth Table

充分性分析從觀察到條件出發，往前看結果是否在場。

指標公式（清晰集）意義
充分性 Inclusion
InclS		O=1 且 C=1 ÷ 所有 C=1C 在場時，O 也在場的比例；越高代表越充分
充分性 Coverage
CovS		O=1 且 C=1 ÷ 所有 O=1C 能解釋多少 O=1 的案例；越高代表解釋力越強
唯一涵蓋率
cov.u		只被該 PI 解釋、其他 PI 未覆蓋的部分各解項的獨特貢獻度

真值表 · Truth Table

k 個條件產生 2k 個「組態 (configuration)」。truthTable() 為每個有案例的組態計算 OUT（outcome value）：

OUT 值意義條件
1這個組態「充分」導致結果inclusion ≥ incl.cut1（預設 1.0）
0這個組態不充分inclusion < incl.cut0
C矛盾 (contradiction)介於 incl.cut0 與 incl.cut1 之間
?邏輯餘項 (logical remainder)案例數 < n.cut（不夠多案例）
> KrookTT <- truthTable(Krook, outcome = "WNP", + show.cases = TRUE, sort.by = c("incl", "n")) ES QU WS WM LP OUT n incl PRI cases 28 1 1 0 1 1 1 2 1.000 1.000 NL,BE 32 1 1 1 1 1 1 2 1.000 1.000 NO,IS 4 0 0 0 1 1 1 1 1.000 1.000 NZ ... 11 0 1 0 1 0 0 4 0.000 0.000 AU,GB,FR,IE 25 1 1 0 0 0 0 3 0.000 0.000 CH,PT,GR
互動 OUT 值判定邏輯模擬器

輸入一個組態的案例數與 inclusion score，看 truthTable() 如何決定 OUT 值。

3.4 布林最小化：三種解 · Boolean Minimization: Three Solutions

把真值表中所有 OUT=1 的組態做「布林最小化」，就能得到精簡的解。eqmcc()（enhanced Quine-McCluskey）是 QCA 套件的核心函式。

三種解的關係

這三種解從「保守」到「簡約」，是一個連續的光譜，差別在於邏輯餘項 (logical remainders)是否以及如何被納入最小化：

複雜解 · Complex Solution

> KrookSC <- eqmcc(KrookTT, details = TRUE, show.cases = TRUE) S1: ES*QU*ws*LP + ES*QU*ws*WM + es*ws*WM*LP + ES*WS*wm*lp + ES*WS*WM*LP incl PRI cov.r cov.u cases 1 ES*QU*ws*LP 1.000 1.000 0.273 0.091 NL,BE; AT 2 ES*QU*ws*WM 1.000 1.000 0.273 0.091 NL,BE; ES ... S1 1.000 1.000 1.000

簡約解 · Parsimonious Solution

> KrookSP <- eqmcc(KrookTT, include = "?", rowdom = FALSE, details = TRUE) S1: WS + ES*WM + QU*LP + (es*LP) S2: WS + ES*WM + QU*LP + (WM*LP) ← rowdom=TRUE 預設會選 S2（WM*LP 支配 es*LP）
什麼是「行支配 row dominance」？

當兩個非必要 PI 中，P1 所涵蓋的所有組態 P2 也涵蓋，但 P1 額外涵蓋更多組態時，P2 被 P1「支配」。預設 rowdom = TRUE 會自動去除被支配的 PI，讓解更精簡。

中間解 · Intermediate Solution

> KrookSI <- eqmcc(KrookTT, include = "?", + direxp = rep(1, 5), # 5個條件都預期出現=1時有利 + details = TRUE) p.sol: WS + ES*WM + QU*LP + WM*LP ← 來自哪個簡約解 S1: ES*WS + WM*LP + ES*QU*LP + ES*QU*WM incl PRI cov.r cov.u 1 ES*WS 1.000 1.000 0.455 0.182 2 WM*LP 1.000 1.000 0.636 0.182 3 ES*QU*LP 1.000 1.000 0.455 0.091 4 ES*QU*WM 1.000 1.000 0.455 0.091 S1 1.000 1.000 1.000
如何解讀 POF 表

每個 PI（Prime Implicant）都有四個指標：incl（consistency，夠高代表充分）、PRI（消除矛盾後的 inclusion）、cov.r（原始涵蓋率）、cov.u（唯一涵蓋率）。最後一行是整個解的總體指標。唯一涵蓋率不適用於整個解，故不顯示。

概念圖 三種解的比較摘要
解的類型eqmcc() 關鍵參數使用邏輯餘項？特點建議用途
複雜解include = "1"（預設）否最長、最保守，每個條件都完整完整性基準
簡約解include = "?"全部用最短，可能含理論不合理的假設設定 direxp 的起點
中間解include="?" + direxp只用符合期待的在簡約與複雜之間，理論上最有說服力正式報告首選

3.5 矛盾性化簡假設 CSA · Contradictory Simplifying Assumptions

當某個邏輯餘項同時被用來化簡「結果出現（Y=1）」與「結果不出現（Y=0）」的解時，就構成矛盾性化簡假設 (Contradictory Simplifying Assumption, CSA)。CSA 代表這個餘項的「反事實」立場在邏輯上是自相矛盾的，應謹慎報告。

# Step 1：對否定結果做簡約解 > KrookSPn <- eqmcc(Krook, outcome = "WNP", neg.out = TRUE, include = "?") # Step 2：取正解與負解各自 SA 的交集 > SAs_neg <- rownames(KrookSPn$SA$S1) > SAs_pos <- rownames(KrookSP$SA$S2) > (CSA <- intersect(SAs_neg, SAs_pos)) [1] "5" "7" "13" "15" > KrookTT$tt[CSA, ] # 找出這些組態是什麼

3.6 繪圖：Venn 圖 · Plotting Venn Diagrams

Venn 圖是 csQCA 視覺化的最佳工具（注意：模糊集不能用 Venn 圖）。用 VennDiagram 套件，搭配 which() 函式將隸屬度資料轉成 Venn 所需的案例編號清單：

> library("VennDiagram") > PIms <- KrookSI$pims$i.sol$C1P1 # 取中間解的 PI 隸屬度 > vennPlot <- venn.diagram( + x = list( + "WNP" = which(Krook$WNP == 1), + "ES*WS" = which(PIms[ , 1] == 1), + "WM*LP" = which(PIms[ , 2] == 1), + "ES*QU*LP"= which(PIms[ , 3] == 1)), + filename = NULL) > grid.draw(vennPlot)
VennDiagram 套件限制

此套件最多支援 4 個集合（order 4），超過就難以清晰呈現。filename = NULL 時直接在 R 繪圖裝置顯示；設定檔名則儲存為 .tiff；用 NULL 再另存才能輸出其他格式（.pdf/.png/.jpg）。

本章回顧

csQCA 完整流程

① 校準：ifelse()／calibrate() 把原始變數轉成 0/1 集合。
② 必要性：superSubset() 搜尋 InclN 與 CovN 夠高的必要條件。
③ 真值表：truthTable() 建立 2^k 組態表，每個組態得到 OUT（1/0/C/?）。
④ 最小化：eqmcc() 推導複雜解（只用有案例組態）、簡約解（加邏輯餘項）、中間解（加方向期待）。
⑤ CSA 診斷：intersect() 找出正解與負解的共同 SA。
⑥ Venn 圖：which() + venn.diagram() 視覺化必要性或充分性關係。
