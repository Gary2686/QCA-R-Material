4.1 模糊集校準 · Calibrating Fuzzy Sets

模糊集（fuzzy set）允許案例的隸屬度連續分布於 0 到 1 之間。隸屬度 1 代表「完全屬於集合」，0 代表「完全不屬於集合」，而 0.5 是交叉點 (crossover point)——最大不確定性，既非屬於亦非不屬於。相較於清晰集，模糊集能捕捉真實世界中「程度」的差異，更符合社會科學概念的連續性本質。

本章研究案例

Emmenegger (2011) 分析 18 個富裕民主國家的就業保障法規強度（JSR，outcome）。五個條件均以模糊集校準：
CO（工會集中度）、UR（工會權利）、SP（社會伙伴關係）、UE（失業率）、LP（左翼政黨執政比例）。

交叉點的重要性 · The crossover point

交叉點（cross = 0.5）是模糊校準中最關鍵的決策點。案例隸屬度 > 0.5 代表「更屬於集合」，< 0.5 代表「更不屬於集合」。交叉點必須有實質的理論或概念依據——它不是統計中位數，而是「概念上模糊不清」的那個邊界值。

轉換校準法：calibrate() · Transformational calibration

QCA 套件的 calibrate() 函數將連續原始值透過邏輯斯蒂函數（S 型曲線）轉換為模糊隸屬度。需指定三個閾值：

參數名稱概念意義對應輸出隸屬度
e.ex完全排除閾值
(fully-out threshold)		原始值低於此處，案例幾乎完全不屬於集合≈ 0.05
cross交叉點
(crossover point)		概念上「半屬於」的那個原始值；最大不確定性= 0.5
e.in完全納入閾值
(fully-in threshold)		原始值高於此處，案例幾乎完全屬於集合≈ 0.95

以就業保障法規（JSR，指數 1–10）為例，研究者認定：指數 ≥ 8 表示高保障（完全屬於），指數 = 5 是「說不準」（交叉點），指數 ≤ 2 表示低保障（完全不屬於）：

> library(QCA) > data(Emmenegger) # 載入 Emmenegger 資料集 > Emm <- Emmenegger # 工作副本 > # 校準 JSR：正端點（遞增型），e.ex=2, cross=5, e.in=8 > Emm$JSR <- calibrate(Emmenegger$JSR, + type = "fuzzy", + thresholds = c(2, 5, 8)) # c(e.ex, cross, e.in) > # 或用有名字的字串寫法： > Emm$JSR <- calibrate(Emmenegger$JSR, + type = "fuzzy", + thresholds = "e.in=8, cross=5, e.ex=2") > round(Emm$JSR, 3)
thresholds 向量順序

以向量傳入時，QCA 按升序排列對應 e.ex → cross → e.in（正端點，隸屬度遞增）；若傳入降序（如 c(8, 5, 2)），則對應 e.ex → cross → e.in（負端點，隸屬度遞減）。排列順序決定校準的方向，務必搭配理論含義確認。

四種概念類型 · Four calibration concept types

依據閾值排列方式，模糊集校準描述四種不同的概念形態。前兩種（端點型）最為常見：

S1 — 正端點（Endpoint-Positive）

原始值愈高 → 隸屬度愈高。最常見。
排列：e.ex < cross < e.in

範例：左翼政黨執政比例 → 「左翼主政」集合

S2 — 負端點（Endpoint-Negative）

原始值愈高 → 隸屬度愈低（取補集）。
排列：e.ex > cross > e.in

範例：失業率 → 「低失業率」集合（高失業 = 低隸屬）

M1 — 正中點（Midpoint-Positive）

中間值隸屬度最高，兩端低。需 4 個閾值：
c(e.ex.lo, e.in.lo, e.in.hi, e.ex.hi)

範例：溫度 → 「宜居溫度」集合（極端皆低隸屬）

M2 — 負中點（Midpoint-Negative）

中間值隸屬度最低，兩端高。概念的典型在兩極。
需 4 個閾值。

範例：政黨意識形態 → 「極端」集合（中間派低隸屬）

互動 模糊校準曲線模擬器

調整三個閾值，觀察 S 型校準曲線如何變化，以及原始值 1–10 被轉換成什麼隸屬度。遞增排列（e.ex < cross < e.in）為 S1 正端點；遞減排列為 S2 負端點。

ECDF 校準法 · ECDF calibration

當研究者缺乏明確的理論閾值時，可用「經驗累積分布函數 (Empirical CDF, ECDF)」校準：以資料的分位數位置作為隸屬度，確保隸屬度分布反映資料本身的分布形狀。

ECDF 校準的限制

ECDF 校準沒有真正的概念型「交叉點」，其分位數 0.5 不代表集合理論上的模糊邊界，只是分布的中位數。Ragin (2008) 建議只在有明確理論支撐的三點閾值無法取得時才考慮 ECDF，並需在報告中說明理由。

> # ECDF 校準方法一：用 R base 的 ecdf() > ecdf_fn <- ecdf(Emmenegger$LP) > Emm$LP_ecdf <- ecdf_fn(Emmenegger$LP) > > # ECDF 校準方法二：用 calibrate() 搭配 quantile() 自動計算閾值 > q_thresh <- quantile(Emmenegger$LP, c(0.1, 0.5, 0.9)) > Emm$LP <- calibrate(Emmenegger$LP, + type = "fuzzy", + thresholds = q_thresh)

4.2 模糊集必要性 · Necessity in fsQCA

在清晰集 csQCA 中，必要性是二元的：條件 C 要麼覆蓋所有結果為 1 的案例，要麼沒有。在模糊集 fsQCA 中，必要性是程度問題：我們用「最小值 (min) 運算」來量化 O 在多大程度上是 C 的子集。

集合論直覺 · Set-theoretic logic

「C 是 O 的必要條件」在集合論中等同於「O ⊆ C（O 是 C 的子集）」。在模糊集中，O ⊆ C 意味著每個案例的 Oi ≤ Ci。當存在違反此規則的案例時，一致性 (InclN) 會下降：

公式 · Formulas

InclN（必要性一致性）= Σ min(Ci, Oi) / Σ Oi
CovN（必要性涵蓋率）= Σ min(Ci, Oi) / Σ Ci
RoN（必要性相關性）= [Σ Oi − Σ min(Ci, Oi)] / Σ(1−Ci)
min(C, O) = 取兩者中的較小值，代表「同時屬於 C 且屬於 O」的程度（模糊 AND）

三個指標的解讀：

指標計算邏輯含義門檻建議
InclN min(C,O) 之和 ÷ O 之和 「O 在場時，C 也在場」的程度。越高越支持必要性。 ≥ 0.90
CovN min(C,O) 之和 ÷ C 之和 C 是否「太過寬泛（trivially necessary）」。越高表示 C 非瑣碎。 ≥ 0.60
RoN （O之和 − min之和）÷ (1−C) 之和 C 對解釋 ~O（結果缺席）的相對貢獻。高 RoN 警示 C 幾乎是常數 1。 > 0.50 為可接受

PRI：比例一致性減量 · Proportional Reduction in Inconsistency

PRI 用來偵測「同時必要且充分 (SUIN/NOSC)」的特殊問題——當 C 同時支持 O 和 ~O（結果與其補集），InclN 可能被高估。PRI 修正了這個問題：

PRI = [ Σ min(C,O) − Σ min(C, O, ~O) ] / [ Σ O − Σ min(C, O, ~O) ]
PRI → 1：C 幾乎只支持 O（好）
PRI → 0：C 同時強烈支持 O 和 ~O（SUIN 問題，需注意）

使用 superSubset() 和 pof()

> # 必要性分析：搜尋 JSR 的必要條件 > nec_result <- superSubset(Emm, + outcome = "JSR", + conditions = c("CO", "UR", "SP", "UE", "LP"), + incl.cut = 0.9, # InclN 門檻 + cov.cut = 0.6, # CovN 門檻（非瑣碎） + ron.cut = 0.5) # RoN 門檻 > nec_result ## incl PRI cov.r cov.u RoN ## SP 0.910 0.858 0.730 0.730 0.713 ## CO 0.882 0.821 0.691 0.691 0.743 > # pof() 手動計算特定條件的適合度指標 > # pof(setx, sety) 計算 setx ⊆ sety 的充分性（setx 為子集） > # → 互換 setx/sety 即得到必要性（sety ⊆ setx） > pof(Emm$JSR, Emm$SP) # JSR ⊆ SP → SP 作為 JSR 的必要條件 ## incl PRI cov.r cov.u ## JSR 0.910 0.858 0.730 0.730

4.3 充分性與真值表 · Sufficiency & Truth Table

fsQCA 的充分性分析同樣以真值表為核心，但建構方式與 csQCA 有兩點不同：（1）案例用 >0.5 規則分配到真值表列（binarization）；（2）計算每列 InclS 時仍使用原始模糊分數（而非二元化後的值）。

大於 0.5 規則 · The >0.5 rule

每個案例依其 k 個條件的模糊隸屬度決定所屬的真值表列：

條件 Ci > 0.5 → 該列對應的位元為 1（案例「更屬於」此條件集合）
條件 Ci ≤ 0.5 → 該列對應的位元為 0（案例「更不屬於」此條件集合）
例：案例 A 的 (CO=0.71, UR=0.29, SP=0.83, UE=0.44, LP=0.62)
→ 二元化為 [1, 0, 1, 0, 1] → 分配到真值表第 21 列（10101₂ = 21）
為何要先二元化？

fsQCA 的 2^k 真值表結構來自集合論的組態思維——每個條件「在場」或「缺席」構成一種組態，對應確定的因果解釋。二元化讓研究者能判斷哪些組態有足夠的案例支持，以及每種組態是否充分導致結果。

InclS 與 CovS 的模糊公式 · Fuzzy InclS and CovS

Xi = min(C1i, C2i, …, Cki)　（列 i 的模糊 AND，所有條件隸屬度的最小值）
InclS = Σ min(Xi, Oi) / Σ Xi　（充分性一致性）
CovS = Σ min(Xi, Oi) / Σ Oi　（充分性涵蓋率）
CovU = 唯一涵蓋率　（排除其他路徑後，此路徑獨立貢獻的涵蓋率）

建構真值表 · Building the truth table

> tt <- truthTable(Emm, + outcome = "JSR", + conditions = c("CO", "UR", "SP", "UE", "LP"), + incl.cut = 0.8, # InclS 門檻（fsQCA 常用 0.75–0.85） + n.cut = 1, # 每列至少 1 個案例才納入 + complete = TRUE, # 顯示所有 2^5=32 列（含邏輯餘項） + show.cases = TRUE, # 顯示各列包含哪些案例 + sort.by = "incl") # 依 InclS 由高至低排序 > tt # 檢視真值表 > tt$tt # 直接存取真值表 data.frame > # 查看 OUT 編碼邏輯 > # OUT=1：incl ≥ incl.cut；OUT=0：incl < incl.cut；OUT=?：邏輯餘項
fsQCA 的 incl.cut 如何選擇？

fsQCA 的一致性門檻反映的是「模糊充分性」的接受水準：
• Ragin (2008) 建議最低 0.75
• 實務研究常採用 0.80 作為基準
• 高度嚴格的研究可設到 0.85–0.90
門檻越高，進入最小化的列越少（越嚴格），但可能遺漏弱充分的路徑。應在複雜解、簡約解、中間解三者中保持一致。

三種解的推導 · Deriving three solutions

> # ── 複雜解（Conservative Solution）────────────────────────────── > # 只用有案例的組態（OUT=1），不化簡邏輯餘項 > cs <- eqmcc(tt, details = TRUE, show.cases = TRUE) > cs > # ── 簡約解（Parsimonious Solution）────────────────────────────── > # 加入所有邏輯餘項（include="?"），讓布林最小化自由化簡 > ps <- eqmcc(tt, + include = "?", + details = TRUE, + show.cases = TRUE) > ps > # ── 中間解（Intermediate Solution）────────────────────────────── > # 加入邏輯餘項，但只用符合「方向性預期」的邏輯餘項 > is <- eqmcc(tt, + include = "?", + direxp = c(CO=1, UR=1, SP=1, UE=0, LP=1), + details = TRUE, + show.cases = TRUE) > is
方向性預期 direxp 的設定依據

direxp 中 1 表示「預期條件以在場形式進入解（正向）」，0 表示「以缺席（取補集 ~）形式進入解」。以本例而言：研究者理論上預期 CO（工會集中度高）、UR（工會權利多）、SP（有社會伙伴關係）、LP（左翼政黨執政比例高）均「在場」才有利於高就業保障；而 UE（高失業）「缺席」（低失業）才有利。設定 direxp 必須有文獻或理論根據，而非資料導向。

矛盾性化簡假設 CSA 檢驗 · Checking for CSA

中間解與簡約解都使用了「化簡假設 (Simplifying Assumptions, SA)」——假設某邏輯餘項的結果為 1，從而允許化簡。若某一邏輯餘項同時被假設為 1（在簡約解的正向分析中）和 0（在簡約解的負向分析中），則構成「矛盾性化簡假設 (Contradictory Simplifying Assumptions, CSA)」：

> # 取得三種解各自使用的化簡假設清單 > SA_is <- is$SA # 中間解的 SA（包含 $pos 和 $neg） > SA_ps <- ps$SA # 簡約解的 SA > # CSA = 中間解中，同時出現於正向與負向 SA 的邏輯餘項 > CSA <- intersect(SA_is$pos, SA_is$neg) > CSA ## character(0) # 理想情況：無 CSA > # 若有 CSA，可查看對應的組態 > if(length(CSA) > 0) tt$tt[CSA, ]
互動 真值表列分配模擬器

輸入一個案例的 5 個條件模糊隸屬度，觀察它被 >0.5 規則分配到哪個真值表列，以及列的 min 值（模糊 AND）。

4.4 繪圖：XY 散佈圖 · XY Scatter Plots

XY 散佈圖（又稱集合論散佈圖）是 fsQCA 最重要的視覺化工具：透過案例在二維空間的分布，直觀呈現集合關係（必要性或充分性）的一致性程度。

必要性 XY 圖 · Necessity XY plot

橫軸（X）= 條件 C 的模糊隸屬度；縱軸（Y）= 結果 O 的模糊隸屬度。若 C 是 O 的完美必要條件，所有案例應落在對角線 Y = X 以下（O ≤ C）。

充分性 XY 圖 · Sufficiency XY plot

橫軸（X）= 條件 C；縱軸（Y）= 結果 O。若 C 是 O 的完美充分條件，所有案例應落在對角線以上（O ≥ C）。

> # ── 必要性 XY 圖：SP 是否為 JSR 的必要條件？ ── > plot(Emm$SP, Emm$JSR, + xlim = c(0, 1), ylim = c(0, 1), + xlab = "SP (Social Partnership)", + ylab = "JSR (Job Security Regulations)", + main = "Necessity plot: Is SP necessary for JSR?", + pch = 21, bg = "steelblue", cex = 1.2) > abline(a = 0, b = 1, lty = 2, col = "red") # 對角線 > text(Emm$SP, Emm$JSR, + labels = rownames(Emm), pos = 3, cex = 0.65) # 案例標籤 > > # ── 充分性 XY 圖（SP → JSR）── > plot(Emm$SP, Emm$JSR, + xlim = c(0, 1), ylim = c(0, 1), + xlab = "SP", ylab = "JSR", + main = "Sufficiency plot: Is SP sufficient for JSR?", + pch = 21, bg = "darkorange", cex = 1.2) > abline(a = 0, b = 1, lty = 2, col = "red") > > # ── 解的路徑 XY 圖（用最小化後的 PI 隸屬度）── > # is$pims$i.sol$C1P1 包含每個 PI 對每個案例的模糊隸屬度 > PI_memb <- is$pims$i.sol$C1P1[ , 1] # 取第一條路徑的隸屬度 > plot(PI_memb, Emm$JSR, + xlim = c(0, 1), ylim = c(0, 1), + xlab = "PI membership (path 1)", ylab = "JSR", + main = "Sufficiency: Path 1 of intermediate solution") > abline(a = 0, b = 1, lty = 2, col = "red")
讀圖要訣

必要性圖：案例點分布在對角線以下代表 C 覆蓋 O（O ≤ C）；落在對角線左上方的點是「違反案例」——O 高但 C 低，拉低 InclN。
充分性圖：案例點應在對角線以上（O ≥ C）；落在右下方的點是違反充分性的案例。
兩種圖的對角線斜率都是 1，截距都是 0（abline(a=0, b=1)）。案例離線越遠，對應的一致性貢獻越差。

本章回顧 · Chapter Summary

fsQCA 完整分析流程
  1. 校準：calibrate(type="fuzzy", thresholds=c(e.ex, cross, e.in))。三個閾值必須有理論依據，特別是交叉點（cross）。四種概念類型：S1（正端點）、S2（負端點）、M1（正中點）、M2（負中點）。
  2. 必要性：superSubset(incl.cut=0.9, cov.cut=0.6, ron.cut=0.5)。評估 InclN、CovN、PRI、RoN 四個指標；PRI 低表示 SUIN 問題。
  3. 真值表：truthTable(incl.cut=0.8, n.cut=1)。用 >0.5 規則分配案例到 2^k 列；InclS ≥ 0.8 列編碼 OUT=1。
  4. 布林最小化：eqmcc() 複雜解；include="?" 得簡約解；direxp 加入方向性預期得中間解。
  5. CSA 診斷：intersect(SA_is$pos, SA_is$neg) 找矛盾化簡假設；理想情況為空集合。
  6. 視覺化：plot() + abline(a=0, b=1) 畫必要性圖（點應在線下）與充分性圖（點應在線上）。
比較面向csQCA（清晰集）fsQCA（模糊集）
隸屬度0 或 1（二元）0.001 到 0.999（連續）
校準函數calibrate(thresholds=t)
或 ifelse()		calibrate(type="fuzzy",
thresholds=c(e.ex, cross, e.in))
AND 運算最小值 min() ≡ 乘積（csQCA 下等同）最小值 pmin() 或 min()
真值表列分配直接用 0/1 隸屬度>0.5 規則二元化
InclS 計算比例（出現 1 的案例數）Σmin(X,O)/ΣX（連續加權）
incl.cut 常用值1.0（完全一致）0.75–0.85
PRI不適用偵測同時支持 O 和 ~O 的問題
視覺化Venn 圖（VennDiagram 套件）XY 散佈圖（plot + abline）
